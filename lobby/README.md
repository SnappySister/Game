# 游戏大厅 - 局域网多人游戏平台

基于 Node.js + WebSocket + 原生 HTML/JS 构建的局域网多人游戏大厅。支持玩家通过浏览器快速进入，在大厅选择游戏、创建或加入房间，与好友实时对战或观战。

平台采用模块化架构设计，每个游戏以独立引擎的形式接入，未来可不断扩展新游戏。

## 功能特性

### 大厅系统
- **昵称登录**：无需注册，输入昵称即可进入大厅
- **游戏列表**：大厅展示平台所有游戏，点击即可进入对应游戏区
- **房间系统**：每个游戏区内可创建/加入/离开房间，支持观战正在进行的对局
- **双频道聊天**：
  - 大厅频道：所有在线玩家可见
  - 房间频道：仅限房间内玩家和观战者
  - 消息独立存储，切换标签互不干扰
  - 未读消息红点提醒（标签 + 浮动按钮）

### 房间功能
- **创建房间**：自定义房间名称，成为房主
- **加入房间**：选择等待中的房间加入对战
- **观战模式**：进入已满员或进行中的房间，实时观看战局
- **准备/取消准备**：玩家就位后点击准备，房主确认全员准备即可开始
- **结束对局**（房主）：游戏中房主可强制结束对局，所有人回到准备状态

## 已上线游戏

### 卡牌对战（2人）
回合制卡牌对战，双方轮流出牌，通过策略击败对手。

核心机制：
- **水晶系统**：每回合金上限 +1（最高 10），回合开始时回满
- **Weighted 抽牌**：费用越高的牌抽到概率越低
- **后手补偿**：后手玩家开局获得水晶硬币（0费+1水晶）
- **手牌上限**：非硬币手牌数不能超过当前水晶上限
- **负面效果**：中毒、灼烧、冰冻、死亡倒计时
- **免疫机制**：格挡免疫伤害，护罩免疫负面效果

- **冰冻**：该回合无法出牌，只能打出净化解除，1 回合后自动融化
- **死亡倒计时**：每回合 -1，归零爆发扣血，可用净化解除

### 德州扑克（2-6人）
多人德州扑克（无限注），通过策略和牌型大小赢取筹码。

核心机制：
- **筹码**：每人初始 1000 筹码，跨局连续保留
- **盲注**：小盲 10，大盲 20，每局庄家顺时针轮换
- **发牌**：每人 2 张底牌（仅自己可见），公共牌分三轮发出：翻牌 3 张、转牌 1 张、河牌 1 张
- **下注轮**：Preflop → Flop → Turn → River
- **操作**：弃牌 / 过牌 / 跟注 / 加注 / 全下
- **比牌**：未弃牌玩家用 2 张底牌 + 5 张公牌选出最大的 5 张组合比大小
- **边池分配**：allin 玩家自动创建边池，按 `<总下注额度 × 参与人数>` 结算
- **继续下一局**：所有人投票确认后才继续（如果有人离开，按剩余在场人数计算）

牌型大小（从大到小）：同花顺 > 四条 > 葫芦 > 同花 > 顺子 > 三条 > 两对 > 一对 > 高牌

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Node.js + `ws` (WebSocket) |
| 前端 | 原生 HTML5 + CSS3 + JavaScript（单页应用） |

```
│   └── style.css               # 暗黑风格样式
└── test-*.js                   # 测试脚本（引擎级 + WebSocket 端到端）
```

## 安装与运行

### 本地运行
```bash
cd lobby
npm install
node server.js
```
浏览器打开 `http://localhost:3456`

### 局域网联机
1. 查看本机局域网 IP：
   ```bash
   ipconfig | findstr IPv4
   # 例如 192.168.1.5
   ```
2. 确保防火墙放行 3456 端口（Windows）：
   ```bash
   netsh advfirewall firewall add rule name="GameLobby" dir=in action=allow protocol=tcp localport=3456
   ```
3. 同局域网内其他玩家访问：
   ```
   http://192.168.1.5:3456
   ```

## 开发与扩展

### 添加新游戏
平台采用插件式架构，接入新游戏只需三步：

1. 在 `games/` 目录下新建游戏引擎类
   - 构造函数接收玩家列表和消息发送回调
   - 提供 `start()`、`handleMessage(playerIndex, msg)` 接口
   - 参考 `card-game.js` 或 `texas-holdem.js` 的实现范式

2. 在 `server.js` 的 `GAMES` 数组中注册游戏
   ```js
   { id: 'snake', name: '贪吃蛇对战', desc: '多人实时蛇蛇大作战', minPlayers: 2, maxPlayers: 4, available: true }
   ```

3. 在前端 `index.html` 中添加对应的游戏选择和渲染逻辑

### 修改现有游戏
卡牌数值、效果等配置直接修改 `lobby/games/card-game.js` 顶部的 `CARDS` 数组即可，无需重启服务器即可生效（新房间会使用新配置）。
